Architecture maps

Core model

Three declarative layers

Machine setup is three hand-maintained sources of truth, each applied by a tool this repository does not implement. The repo scripts only sequence them; there is no custom install engine.

Layers and owners

flowchart LR
  Brewfile[Brewfile / Brewfile.posix] --> BrewBundle[brew bundle]
  Trees[Stow trees: zsh/, git/, nvim/, mise/, ...] --> StowSh[./stow.sh]
  StowSh --> MiseConfig[~/.config/mise/config.toml]
  MiseConfig --> MiseInstall[mise install]
  BrewBundle --> Machine[(machine state)]
  StowSh --> Machine
  MiseInstall --> Machine

Ordering matters once: the global mise config reaches $HOME through the mise/ Stow tree, so stow must run before mise install on a fresh machine. bootstrap.sh encodes that sequence.

brew bundle

Owns macOS-only concerns: casks, fonts, Mac App Store apps, and formulae that are better from Homebrew. Brewfile.posix covers Homebrew on Linux.

Brewfile.posix

stow

Symlinks dotfile packages into $HOME. STOW_DIRS in stow.sh is the package registry; a stow-only machine (e.g. work) needs nothing else.

stow.sh

mise install

Owns cross-platform CLI tools and language runtimes. One TOML file installs the same tools on macOS and Linux, via registry short names or github:owner/repo backends.

mise/.config/mise/config.toml

Deliberate exclusions

Unmanaged tools. AI CLIs (claude, codex, opencode, copilot) self-update through their native installers. Adding them to a layer would fight the updater, so their absence is a decision, not a gap.

No generators. The Brewfiles and the mise config are hand-maintained. The previous YAML catalog, manifest generators, and 2,400-line install library were deleted in July 2026; git log has the history.

Drift detection

make audit (scripts/audit-installed.sh) reports Brewfile entries missing from the machine, installed leaves/casks missing from the Brewfile, and mise tool status — keeping the declared and actual state honest in both directions.
