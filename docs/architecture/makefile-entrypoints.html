Entrypoints and orchestration

Four small entrypoints sequence the three declarative layers. None of them owns package or symlink logic; they call brew bundle, ./stow.sh, and mise install in the right order for their audience.

Who runs what

Fresh Mac

./setup-personal-mac.sh — bootstrap + macOS settings + SSH from 1Password, or ./bootstrap.sh alone for just the layers.

Day to day

make install (re-apply all layers), make update (brew + mise + mas), make configure (macOS defaults, MACOS_PROFILE=personal|work).

Stow-only machine

Work machines run git clone + ./stow.sh. No package management crosses that boundary.

Important seams

STOW_DIRS (stow.sh)

The package registry. New dotfile trees are added here, nowhere else.

BREW_WITH_POLICY

All Makefile brew calls go through the tap-trust policy adapter, keeping policy out of targets.

BREW_UPDATE

Adapter for the Homebrew update sequence, keeping the Makefile from owning ordering.

CLI contract

Every user-facing entrypoint keeps --help with examples, --dry-run when it mutates, and non-interactive escapes (--no-input, skip flags). _test/cli-contracts.bats enforces this.
