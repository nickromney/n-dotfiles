Architecture maps

Entrypoint module

Makefile profile and action router

The Makefile is the first seam most callers cross. Its interface is profile plus action, while its implementation selects config bundles and delegates to deeper shell modules.

Interface and implementation

Interface

Profiles: common, personal, work, all. Actions: install, update, stow, configure. Focus areas are direct config adapters.

Implementation

Make variables compute PROFILE_CONFIGS, SELECTED_BREWFILE, and SELECTED_MACOS_PROFILE, then shell modules perform the actual work.

Depth

Callers learn one command shape; package-manager and host-state details stay behind seams.

Important seams

PROFILE_CONFIGS

Maps profile words to ordered Configuration Bundles. Shared bundles come first.

FOCUS_AREAS

Bypasses profile selection and sends one bundle directly to install.sh.

BREW_UPDATE

Adapter for full Homebrew update contexts, keeping the Makefile from owning the sequence.
