Host state modules

macOS, setup, and Stow flows

Package installation, macOS defaults, and dotfile symlinks are separate modules with separate interfaces. The setup scripts compose them for fresh or profile-specific machines.

Fresh host flow

flowchart LR
  Bootstrap[bootstrap.sh] --> Brew[Homebrew + brew bundle]
  Brew --> Stow[./stow.sh]
  Stow --> Mise[mise install]
  Mise --> Macos[make configure]
  Macos --> SSH[setup-ssh-from-1password.sh]

Stow Tree

Directories such as zsh/, git/, nvim/, and mise/ are modules whose implementation is GNU Stow layout.

Setup Flow

setup-personal-mac.sh composes bootstrap, macOS settings, and SSH setup with skip flags. It owns none of their implementations.

Why this stays separate

Locality. macOS defaults logic lives in _macos/, not in package declarations.

Leverage. ./stow.sh runs alone when packages are irrelevant — it is the entire work-machine setup.

Seam. Setup scripts compose modules but do not own their implementation.
