Execution module

Install and update engine

install.sh is a thin interface. scripts/install-lib.sh holds the implementation: manifest generation, availability detection, manager adapters, update plans, Stow, and reporting.

Execution sequence

sequenceDiagram
  participant User
  participant Install as install.sh
  participant Lib as install-lib.sh
  participant Gen as generate-install-manifests.sh
  participant Managers as manager adapters
  User->>Install: ./install.sh --update -c focus/vscode
  Install->>Lib: parse flags and call main
  Lib->>Gen: generate manifests
  Gen-->>Lib: Brewfile, arkade.tsv, metadata.json
  Lib->>Lib: load metadata and available managers
  Lib->>Managers: run update or install phases
  Lib-->>User: structured Info/Skip/Change output

Install path

Interface. ./install.sh [-c bundle] [-d] [-s] [-f].

Implementation. Generate manifests, run brew bundle, batch arkade get, run direct manager adapters, apply apt fallback, then optionally Stow.

Locality. Manager command details live in manager adapters, not in YAML bundles or Makefile callers.

Update path

Interface. ./install.sh --update updates selected installed tools and skips missing tools.

Implementation. Build update plans from metadata, respect skip_update, write update manifests, and run manager-specific update adapters.

Leverage. One metadata row shape drives Homebrew, arkade, VSCode, mas, cargo, uv, mise, apt, and manual reporting.
