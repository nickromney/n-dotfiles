Harness asset module

Harness asset source and discovery routes

Harness Assets live in source repositories and become visible through harness-specific views. The implementation is intentionally cautious: discovery routes are documented before symlink routes are treated as durable setup behavior.

Asset flow

flowchart LR
  Public[public Harness Repository] --> Shared[Shared Asset Source]
  Private[Optional Private Source] --> Sync[sync-private-harness-assets.sh]
  Shared --> Views[Harness Views]
  Sync --> Views
  Views --> Claude[Claude Code discovery route]
  Views --> Codex[Codex discovery route]
  Views --> Global[Global Skill Root / Chops View]

Shared Asset Source

harnesses/shared/ is the intended public source for reusable Harness Assets. It is not the same as a live discovery route.

Optional Private Source

../harnesses-private may contribute private skills. Absence is a normal zero-change state, not an error.

Harness View

agents/.agents/skills, claude/.claude/skills, and codex/.codex/skills expose assets to Agent Harnesses.

Safety interfaces

sync-private-harness-assets.sh

Interface: --dry-run, --execute, and --private-root. Implementation links selected private skills, detects conflicts, and respects provider load manifests.

audit-harness-guides.sh

Interface: read-only workspace audit. Implementation scans Project Harness Guides for size and repo-local skill references.

Depth rule

A Symlink Route is useful only after a Discovery Route is known. Otherwise the setup flow can create Duplicate Skills or Skill Drift while appearing to install correctly.
